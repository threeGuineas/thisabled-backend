## 1. 서비스 개요

### 1.1 서비스명

**ThisAbled**

### 1.2 서비스 정의 (한 문장)

> 시각·청각·발달장애 사용자가 각자의 장애에 맞춘 인터페이스에서 게시물을 공유하고, AI의 도움을 받아 친구를 만나고, 위험으로부터 보호받으며 소통하는 장애인 맞춤형 AI 소셜 커뮤니티 웹앱이다.

### 1.3 서비스 비전

> 각기 다른 장애를 가진 사용자에게 장애를 극복할 수 있는 맞춤형 인터페이스를 제공하여, 장애를 넘어서는 소통이 이루어지는 커뮤니티를 만든다.

### 1.4 해결하려는 문제

1. 장애인의 사회적 고립과 온라인 관계 형성 기회 부족
2. 기존 소셜 플랫폼의 획일적인 UI로 인한 접근성 장벽
3. 1:1 채팅에서 발생할 수 있는 사기, 성적 접근, 괴롭힘 등의 위험

### 1.5 핵심 사용자 경험

> 사용자가 자신에게 맞는 UI에서 도움 없이 콘텐츠를 이용하고, 잘 맞는 친구를 발견하며, 위험 가능성을 안내받으면서 소통한다.

### 3.1 포함

- 모바일 중심 반응형 웹앱 (PWA)
- 만 14세 이상 가입 (소셜 로그인)
- UI 모드 선택 및 변경
- 시각·청각·발달장애 맞춤형 UI
- 전체 회원 공개 피드
- 텍스트·사진·영상 게시물
- 댓글, 좋아요
- 친구 요청 및 맞춤 친구 추천
- 1:1 채팅 및 비친구 메시지 요청
- AI 안심 채팅 (채팅 텍스트 분석)
- 주의 판정 누적 시 관계 단위 자동 전송 제한
- 미성년자 보호 기본 정책
- AI 소통 코치
- AI 사진 설명 (게시물·채팅)
- 영상 자동 자막 (게시물·채팅)
- 사용자 차단
- 회원 탈퇴

### 3.2 제외 및 후속 범위

- 신고 접수·검토·처리 (접수만 받고 처리하지 못하는 반쪽 기능을 만들지 않기 위해, 관리자 검토 체계와 함께 후속 도입)
- 관리자 화면과 운영자 수동 제재
- 계정 단위 자동 제재 (24시간·7일 채팅 제한, 계정 정지)
- 채팅 사진·동영상 안전 분석
- 실제 장애인 자격 인증
- 그룹 채팅과 관심사 그룹
- 음성·영상 통화
- 복합장애 조합 UI
- 보호자·지원기관 자동 연계
- 비공개 계정과 친구 공개 게시물
- 코칭 레벨 Lv.1~5
- 발달장애인용 그림카드 자동 생성
- 수어 번역 및 수어 검색
- 게시물 임시 저장
- 자막 및 사진 설명 편집

## 4. 사용자 정책

### 4.1 가입 대상

- 최소 가입 연령은 만 14세이다.
- 미성년자와 성인 사이의 친구 관계 및 1:1 채팅을 허용하되, 4.5의 보호 장치를 기본 적용한다.

### 4.2 장애인 자격 확인

- MVP에서는 장애인 자격을 확인하지 않는다.
- 가입 시 장애 유형을 입력받지 않고 사용할 UI 모드만 선택한다.
- 공공기관 API를 이용한 자격 확인은 향후 계획으로 둔다.
- 심사와 시연에는 일반 데모 계정을 사용한다.

### 4.3 복합장애

- 복합장애 전용 조합 UI는 제공하지 않는다.
- 사용자는 한 번에 하나의 모드를 선택한다.
- 복합장애인도 가입할 수 있으며 마이페이지에서 필요한 모드로 변경할 수 있다.

### 4.4 콘텐츠 공개

- 모든 게시물은 가입 회원 전체에게 공개한다.
- 비공개 계정과 게시물별 공개 범위 선택은 제공하지 않는다.
- 차단 관계인 사용자의 게시물과 프로필은 서로 노출하지 않는다.

### 4.5 미성년자 보호 (신설)

> 만 14~18세 사용자를 성인 사용자와의 관계에서 보호하기 위한 기본 정책이다.

- 미성년자 여부는 가입 시 생년월일로부터 서버가 계산한 만 나이를 기준으로 판정한다 (MATCH-02-5 참조).
- **비친구 메시지 요청 기본 차단**: 미성년 사용자는 성인 사용자로부터의 비친구 메시지 요청을 기본으로 받지 않는다. 이 설정은 미성년 사용자 본인이 마이페이지에서 변경할 수 있다.
- **친구 추천 상호 제외**: 맞춤 친구 추천에서 미성년 사용자와 성인 사용자를 서로 추천하지 않는다.
- **민감 판정 기준 적용**: 미성년 사용자가 수신하는 메시지의 `주의` 판정 임계값은 성인보다 낮게(민감하게) 설정한다. 임계값은 모델 평가 결과에 따라 운영 설정값으로 관리한다.
- **미디어 전송 제한**: 채팅 사진·동영상은 AI 안전 분석 대상이 아니므로(SAFE-02), 미성년 사용자와 성인 사용자 간 채팅에서는 사진·동영상 전송을 제한한다 (텍스트만 허용).
- **알려진 한계**: 연령 판정은 가입 시 자진 신고한 생년월일에 기반하며, MVP에는 별도 검증 수단이 없다. 소셜 로그인 제공자의 연령 정보 연동 등 연령 확인 강화는 후속 계획(21장)으로 둔다.

- 선택 모드는 다른 사용자에게 공개하지 않는다.

## 6. 회원가입 및 계정

### ACC-01 회원가입 (소셜 로그인)

> 사용자는 소셜 계정(카카오 우선, 구글 지원)으로 본인 인증 없이 간편하게 가입할 수 있다.

**사용자 시나리오**

1. 사용자가 시작 화면에서 `카카오로 시작하기` 버튼을 실행한다.
2. 카카오 로그인 및 동의 화면을 완료하면 서비스로 돌아온다.
3. 최초 가입자는 추가 정보 입력 화면으로 이동한다: 닉네임, 생년월일, UI 모드, 필수 약관 동의.
4. 만 14세 이상이 확인되면 가입이 완료되고 홈 피드로 이동한다.
5. 선택 정보(프로필 이미지, 자기소개, 관심사 태그)는 가입 직후 또는 마이페이지에서 입력할 수 있다.

**입력값 정의**

| 항목 | 필수 여부 | 제약 | 용도 |
| --- | --- | --- | --- |
| 소셜 계정 (카카오/구글) | 필수 | OAuth 2.0 | 인증·로그인 |
| 닉네임 | 필수 | 중복 불가, 2~12자 | 서비스 내 표시명 |
| 생년월일 | 필수 | 만 14세 이상 | 연령 확인, 미성년 보호, 사용자 매칭 |
| 맞춤 UI 모드 | 필수 | 시각·청각·발달 중 1개 | 화면 구성 |
| 필수 동의 | 필수 | 이용약관 / 개인정보 처리 / 외부 AI 데이터 처리 안내(17장) | 법적 고지 |
| 프로필 이미지 | 선택 | 이미지 1장 | 프로필 |
| 자기소개 | 선택 | 최대 300자 | 프로필, 매칭 입력 |
| 관심사 태그 | 선택 | TAG-01의 체계에서 최대 10개 | 프로필, 매칭 입력 |

**예외 처리**

| 상황 | 처리 |
| --- | --- |
| 만 14세 미만 | 가입을 중단하고 연령 제한을 안내한다 |
| 닉네임 중복 | `이미 사용 중인 닉네임입니다` 표시 후 재입력 |
| 소셜 인증 실패·취소 | 시작 화면으로 복귀하고 재시도를 안내한다 |
| 이미 가입된 소셜 계정 | 가입 절차 없이 즉시 로그인 처리한다 |

- 비밀번호는 수집하지 않는다. 계정 복구는 소셜 계정 제공자의 절차를 따른다.
- 장애인 자격 인증 단계는 MVP에서 표시하지 않는다.
- 닉네임의 허용 문자(한글·영문·숫자 등)와 금칙어 정책은 구현 설계에서 확정한다.

### ACC-02 로그인·로그아웃

- 가입한 소셜 계정으로 로그인하고 로그아웃할 수 있다.
- 세션·토큰 관리 방식은 구현 설계에서 확정한다.

### ACC-03 프로필

- 닉네임, 프로필 이미지, 자기소개 및 관심사를 관리한다.
- 장애 유형과 선택한 UI 모드는 프로필에 표시하지 않는다.
- 자기소개와 관심사는 AI 사용자 매칭의 입력으로 사용한다.

## 7. 피드·게시물·댓글

### FEED-01 홈 피드

> 전체 회원의 게시물을 최신순으로 보여준다.

- 게시물에는 작성자, 작성 시각, 본문, 미디어, 좋아요 수와 댓글 수를 표시한다.
- `공개` 상태의 게시물만 노출한다 (POST-01의 상태 정의 참조).
- 차단 관계의 콘텐츠는 제외한다.
- 발달장애인 모드에서는 한 화면의 정보량을 줄여 표시한다.

### POST-01 게시물 작성

> 사용자는 텍스트, 사진, 영상 게시물을 작성하고 게시 버튼을 실행해 공개한다.

- 텍스트, 사진(최대 3장), 영상(1개)을 지원한다. 음성 전용 게시물은 지원하지 않는다.
- 사용자가 `게시` 버튼을 실행해야만 공개된다. AI 처리 결과가 사용자를 대신해 게시를 실행하지 않는다.
- 게시물 상태는 `작성 중`, `처리 중`, `공개`로 구분한다.
  - 텍스트·사진 게시물: `게시` 실행 즉시 `공개`
  - 영상 게시물: CAPTION-01의 자막 생성 흐름을 따른다. 영상 첨부(업로드) 시점에 게시물 레코드가 `처리 중` 상태의 내부 드래프트로 생성되고, 사용자가 `게시`를 실행하면 `공개`로 전환된다

### POST-02 수정·삭제

- 작성자는 자신의 게시물과 댓글을 수정·삭제할 수 있다.
- 삭제 전 확인 단계를 제공한다.

### POST-03 좋아요·댓글

- 회원은 게시물에 좋아요를 표시하거나 취소할 수 있다.
- 회원은 댓글을 작성할 수 있다.
- 발달장애인 모드에서는 AI 소통 코치의 댓글 추천을 사용할 수 있다.

### VISION-01 AI 사진 설명

> 사진 게시물에 AI가 자동으로 텍스트 설명을 생성하여 시각장애인에게 제공한다.

**입출력 정의**

| 구분 | 내용 |
| --- | --- |
| 입력 | 게시물 첨부 사진 및 채팅으로 전송된 사진 (긴 변 최대 1,280px로 축소 후 분석) |
| 출력 | 사진별 한국어 설명 텍스트, 게시물에 함께 저장 |
| 생성 시점 | 게시물: 사용자가 `게시`를 실행하는 시점 (사진 선택·취소 반복에 의한 중복 호출 방지) / 채팅: 사진 메시지 전송 시점 |

**호출·비용 정책**

- 게시물당 사진 최대 3장
- 사용자당 사진 설명 생성 하루 최대 20회, 분당 최대 5회 (이미지 1장 = 1회, 게시물·채팅 합산)
- 동일 이미지 해시를 캐싱하여 중복 호출 방지
- 실패 재시도 최대 2회
- 서비스 전체 일일 예산과 호출량 상한 설정, 상한 도달 시 자동 호출 중단

**표시 원칙**

- 게시자의 확인을 요구하지 않고 생성된 설명을 게시물에 함께 저장한다.
- 시각장애인에게는 `AI가 생성한 사진 설명`임을 안내한 뒤 설명을 읽는다.
- AI 설명을 원본 사진을 대체하는 사실 정보로 단정하지 않는다.

**채팅 사진 처리**

- 채팅 사진은 전송을 지연시키지 않고 즉시 전달하며, 설명은 생성 완료 후 메시지에 부착한다 (생성 중 `설명 만드는 중` 표시).
- 생성 실패 또는 한도 초과 시 `설명 없음`을 표시하고 메시지 전송은 정상 유지한다.
- 채팅 사진은 안전 분석 대상이 아니다 (SAFE-02).

**예외 처리**

| 상황 | 처리 |
| --- | --- |
| 생성 실패 (재시도 소진) | `사진 설명을 만들지 못했습니다` 표시, 게시물 공개는 정상 진행 |
| 일일·분당 한도 초과 | 설명 없이 게시 가능, 한도 안내 표시 |

### CAPTION-01 영상 자동 자막

> 게시물·채팅에 업로드된 영상의 음성을 자막으로 변환한다. 게시물 영상은 업로드 즉시 자막 생성을 시작하고, 자막이 준비된 뒤 게시할 수 있게 한다.

**사용자 시나리오**

1. 사용자가 게시물 작성 화면에서 영상을 첨부한다.
2. 영상이 서버에 업로드되는 즉시 자막 생성을 시작한다. (업로드 = 일일 자막 생성 횟수 1회 차감)
3. 자막 생성 중에는 작성 화면에 `자막 만드는 중` 상태를 표시한다.
4. 자막 생성이 완료되면 `게시` 버튼이 활성화되고, 사용자가 게시를 실행하면 게시물이 `공개` 상태가 된다.
5. 공개된 게시물에서 시청자는 `자막` 버튼으로 자막 표시를 켜고 끌 수 있다.

**업로드·비용 정책**

| 항목 | 제한 |
| --- | --- |
| 영상 길이 | 최대 3분 |
| 파일 크기 | 최대 200MB |
| 사용자당 업로드 | 하루 최대 5회 (게시물·채팅 합산) |
| 중복 방지 | 동일 영상 해시 캐싱 |
| 실패 재시도 | 최대 2회 |
| 서비스 전체 | 일일 STT 예산 상한, 도달 시 신규 영상 업로드를 보류하고 안내 |

**예외 처리**

| 상황 | 처리 |
| --- | --- |
| 자막 생성 실패 (재시도 소진) | 사용자에게 `다시 시도` 또는 `자막 없이 게시`를 선택하게 한다 |
| 자막 없이 게시 선택 | 게시물에 `자막 없음` 라벨을 표시한다 (청각장애인 모드에서 특히 명확히 안내) |
| 길이·크기 초과 | 업로드를 거부하고 제한을 안내한다 |
| 자막 생성 실패로 재시도가 소진된 경우 | 해당 업로드의 일일 횟수 차감을 복원한다 |

- 자막 생성이 게시를 대신 실행하지 않는다. 공개 여부의 최종 결정은 항상 사용자의 `게시` 실행이다.
- 업로드 후 24시간 내 게시되지 않은 내부 드래프트(POST-01)는 영상·자막과 함께 자동 삭제한다. 이때 이미 성공한 자막 생성의 일일 횟수 차감은 복원하지 않는다.
- MVP에서는 게시자의 자막 편집 기능을 제외한다.

**채팅 영상 처리**

- 채팅 영상은 전송을 지연시키지 않고 즉시 전달하며, 자막은 생성 완료 후 메시지에 부착한다 (생성 중 `자막 만드는 중` 표시).
- 생성 실패(재시도 소진) 시 `자막 없음` 라벨을 표시하고 메시지 전송은 정상 유지한다.
- 길이·크기 제한은 게시물 영상과 동일하다 (최대 3분, 200MB).
- 채팅 영상은 안전 분석 대상이 아니다 (SAFE-02).

## 8. AI 사용자 매칭

### MATCH-01 추천 목적

> 사용자가 소통 가능성이 높은 친구 후보를 발견하도록 돕는다.

- 연애·데이트 매칭이 아니라 커뮤니티 친구 추천이다.

### MATCH-02 추천 입력

추천 모델은 다음 입력을 사용한다.

- 자기소개 텍스트와 관심사 태그
- 사용자의 작성물 및 좋아요 기반 관심 신호
- 공통 친구
- 선택한 UI 모드 (서버 내부 특성으로만 사용)
- 연령대
- 차단 및 기존 친구 관계 (제외 조건)

#### MATCH-02-1 자기소개

- 사용자가 자신을 소개하기 위해 직접 작성하는 선택 입력 항목이다.
- 최대 300자로 제한한다.
- 취미, 선호하는 대화 주제, 성격 및 친구와 하고 싶은 활동 등을 자유롭게 작성할 수 있다.
- 예: `영화와 야구를 좋아해요. 편하게 일상 이야기를 나눌 친구를 찾고 있어요.`
- 자기소개는 프로필에서 다른 사용자에게 공개한다.

#### MATCH-02-2 관심사 태그

- 사용자는 TAG-01의 태그 체계에서 최대 10개의 태그를 선택한다.
- 관심사 태그는 자기소개 텍스트와 별도로 구조화해 저장한다.

#### MATCH-02-3 사용자의 작성물

- 사용자의 게시물 본문, 댓글 텍스트를 Sentence-BERT로 임베딩한다.

#### MATCH-02-4 분석 제외 데이터

다음 데이터는 임베딩과 추천 모델 입력에서 제외한다.

- 닉네임과 프로필 이미지
- 1:1 채팅 메시지
- 전화번호, 이메일 주소 등 연락처
- 정확한 위치
- 장애 유형 인증 정보
- AI 위험 메시지 판정 기록

#### MATCH-02-5 연령 정보

- 가입 시 입력한 생년월일 원문을 추천 모델에 직접 전달하지 않는다.
- 추천 시점의 만 나이와 연령대를 서버에서 계산해 구조화 특성으로 사용한다.
- 연령대는 `14~18세`, `19~24세`, `25~34세`, `35~44세`, `45~54세`, `55세 이상`으로 구분한다.
- 두 사용자의 연령대 일치 여부와 나이 차이를 추천 점수에 반영한다.
- `14~18세`와 성인 연령대는 상호 추천 후보에서 제외한다 (4.5 미성년자 보호).
- 정확한 생년월일은 추천 결과와 추천 사유에 표시하지 않는다.
- 추천 사유가 연령 정보에 기반하면 `비슷한 연령대예요`라고 표시한다.

#### MATCH-02-6 좋아요 기반 관심 신호

- 사용자가 좋아요를 누른 게시물 본문과 댓글 텍스트를 Sentence-BERT로 임베딩한다.
- 사진·영상 자체는 분석하지 않고 해당 게시물의 텍스트와 관심사 정보만 사용한다.
- 좋아요 콘텐츠 임베딩을 결합해 사용자의 암시적 관심 벡터를 생성한다.
- 암시적 관심 벡터는 사용자가 직접 선택한 관심사 태그를 보완하는 신호로 사용한다.
- 추천 결과에 특정 게시물이나 댓글의 원문을 표시하지 않는다.
- 추천 사유는 `관심 있는 콘텐츠가 비슷해요`처럼 일반화해 표시한다.
- 좋아요를 취소하면 해당 콘텐츠를 다음 추천 갱신부터 제외한다.
- 삭제된 콘텐츠, 차단한 사용자의 콘텐츠와 접근할 수 없는 콘텐츠는 즉시 제외한다.
- AI 위험 판정 기록은 좋아요 기반 관심 신호에 포함하지 않는다.
- 분석 대상 기간과 최대 콘텐츠 수는 운영 설정값으로 관리한다.

#### MATCH-02-7 개인정보 입력 방지

- 자기소개에는 전화번호, 이메일, 외부 메신저 ID 입력을 제한한다.
- 연락처 형식이 감지되면 저장 전에 삭제 요청 안내를 표시한다.
- 자기소개 원문은 매칭 목적 외에 사용하지 않는다.

#### MATCH-02-8 미입력 처리

- 자기소개를 작성하지 않아도 서비스를 이용할 수 있다.
- 자기소개가 없으면 관심사, 공통 친구 및 선택한 UI 모드만으로 추천한다.
- 추천 정보가 부족하면 임의의 장애 유형을 추정하지 않고 `추천 정보가 부족합니다`라고 안내한다.

### MATCH-03 추천 모델

- Sentence-BERT로 자기소개와 관심사 텍스트의 의미 유사도를 계산한다.
- LightGBM으로 의미 유사도와 구조화 특성을 결합해 추천 점수를 산출한다.
- 차단 사용자, 이미 친구인 사용자와 요청 거절 이후 30일이 지나지 않은 사용자는 제외한다.
- 추천 사유는 `관심사가 비슷해요`, `공통 친구가 있어요`처럼 이해하기 쉬운 문장으로 표시한다.
- 선택한 UI 모드와 장애 유형 추정값은 추천 사유로 노출하지 않는다.

### MATCH-04 추천 보호 원칙

- UI 모드는 서버 내부 추천 특성으로만 사용한다.
- 추천 결과로 상대방의 장애 유형을 단정하거나 표시하지 않는다.
- 추천 모델 성능은 클릭률뿐 아니라 특정 모드에 추천이 편중되는지도 함께 확인한다.

## 9. 친구·차단

### FRIEND-01 친구 요청

> 사용자는 프로필 또는 맞춤 친구 추천에서 친구 요청을 보내고, 수신자의 수락으로 양방향 친구가 된다.

- 게시물 작성자 프로필 또는 맞춤 친구 추천에서 요청할 수 있다.
- 요청자는 요청을 취소할 수 있다.
- 수신자는 요청을 수락하거나 거절할 수 있다.
- 수락 시 양방향 친구가 된다.

### FRIEND-02 친구 관리

- 친구 목록을 확인하고 친구 관계를 해제할 수 있다.
- 친구가 되면 일반 1:1 채팅을 시작할 수 있다.

### BLOCK-01 사용자 차단

> 차단은 두 사용자 사이의 모든 접점(게시물, 프로필, 요청, 채팅, 추천)을 상호 제거한다.

- 차단 시 기존 친구 관계를 해제한다.
- 서로의 게시물과 프로필을 볼 수 없다.
- 친구 요청, 메시지 요청 및 기존 채팅을 차단한다.
- 맞춤 친구 추천에서 서로 제외한다.

## 10. 1:1 채팅

### CHAT-01 친구 채팅

> 친구 관계인 사용자는 텍스트, 사진, 동영상으로 1:1 채팅을 할 수 있다.

- 그룹 채팅은 제공하지 않는다.
- 텍스트는 AI 안심 채팅의 분석 대상이다 (SAFE-02).
- 사진과 동영상은 MVP에서 안전 분석 대상이 아니며, 대신 사진에는 AI 사진 설명(VISION-01), 동영상에는 자동 자막(CAPTION-01)을 제공한다.
- 사진·동영상 메시지에는 `AI가 안전 분석하지 않은 첨부 파일` 안내를 표시한다.
- 미성년-성인 채팅에서는 사진·동영상 전송을 제한한다 (4.5).

### CHAT-02 비친구 메시지 요청

> 친구가 아닌 사용자의 첫 메시지는 요청함에 분리 보관되며, 수신자의 수락으로 일반 채팅이 된다.

- 친구가 아닌 사용자도 메시지 요청을 보낼 수 있다.
- 요청은 일반 채팅 목록과 분리된 요청함에 표시한다.
- 수신자가 수락해야 일반 채팅방으로 전환된다.
- 수락 전에는 텍스트 메시지 1건만 전송할 수 있다 (사진·동영상 불가).
- 사용자는 마이페이지에서 비친구 메시지 요청 허용 여부를 변경할 수 있다.
- 미성년 사용자는 성인으로부터의 요청이 기본 차단된다 (4.5).
- 차단 또는 연령 보호 정책으로 요청이 차단되는 상대에게는 사유를 표시하지 않고 `요청을 보낼 수 없는 상대입니다`만 표시한다 (미성년 여부 등 수신자 정보 비노출).
- `주의` 판정을 받은 요청 메시지는 요청함에서 블러 상태로 표시한다 (SAFE-03).

### CHAT-03 채팅 상태

- 전송 성공·실패, 분석 상태와 새 메시지 도착 상태를 각 UI 모드에 맞게 표시한다.
- 수신자에게는 방별 미읽음 메시지 수와 전체 미읽음 합계를 제공한다. 미읽음은 해당 사용자가 마지막으로 읽은 위치 이후에 도착한 상대 메시지 수이며, 메시지 목록의 첫 페이지를 열면 그 시점까지 읽음 처리한다.
- 발신자에게는 상대방이 마지막으로 읽은 본인 메시지 한 건에만 읽음 표시를 제공한다. 읽음 상태가 갱신되면 실시간 이벤트로 발신자 화면을 갱신할 수 있다.
- SAFE 재분석으로 보류 메시지가 표시 가능해지는 시점은 새 메시지 도착으로 처리하며, 그 시점 이후 미읽음 수에 포함한다.
- 차단 시 상대방에게 추가 메시지를 보낼 수 없다.

## 11. AI 안심 채팅

### SAFE-01 분석 원칙과 전달 타이밍

> 모든 1:1 텍스트 메시지를 자체 안전 모델로 분석한 뒤 전달한다. 채팅 사진·동영상은 안전 분석 대상이 아니다.

- 텍스트 메시지는 자체 환경에서 운영하는 파인튜닝 한국어 안전 모델로 개별 분석한다. 텍스트 위험 판정에는 외부 AI를 사용하지 않는다.
- 채팅 사진·동영상은 안전 분석 대상이 아니며(SAFE-02), 접근성 처리(사진 설명·자동 자막)는 VISION-01과 CAPTION-01을 따른다.
- 분석 결과는 범죄 확정이 아니라 위험 가능성을 알리는 보조 신호이다.
- 운영자와 다른 사용자는 메시지 원문과 판정 결과를 볼 수 없다.

**전달 타이밍 (관계 무관 동기 처리)**

| 관계 | 전달 방식 | 비고 |
| --- | --- | --- |
| 친구 간 메시지 | 자체 안전 모델의 분석 완료 후 전달 (동기). `주의` 판정이면 블러 상태로 전달 | 자체 모델의 응답이 빠르므로 체감 지연을 최소화한다. 응답 시간 목표는 구현 설계에서 확정 |
| 비친구 메시지 요청 | 분석 완료 후에만 요청함에 표시 (동기). 분석 지연 시 `안전 확인 중` 상태로 보류 | 위험도가 가장 높은 경로의 사전 차단 |

- 분석 완료 후에만 전달하므로 정상 운영 중에는 소급 블러가 발생하지 않는다. 소급 블러는 18.3의 장애 대응(성능 저하 모드)에서만 사용한다.

### SAFE-02 탐지 범위

**분석 대상**

| 대상 | 안전 분석 여부 | 비고 |
| --- | --- | --- |
| 채팅 텍스트 | 분석 | 자체 파인튜닝 안전 모델 |
| 채팅 사진 | MVP 미분석 (후속 계획) | 접근성 목적의 AI 사진 설명(VISION-01)을 제공한다 |
| 채팅 동영상 | MVP 미분석 (후속 계획) | 접근성 목적의 자동 자막(CAPTION-01)을 제공한다 |

**탐지 유형**

- 금전 요구와 사기 가능 표현
- 성적 접근과 만남 강요
- 괴롭힘, 모욕 및 협박
- 탐지 범위를 벗어난 일반적인 사실 판단이나 관계 판단은 수행하지 않는다.

### SAFE-03 사용자 표시

> 위험 가능성이 있는 메시지는 블러 처리하고, 열람 여부의 결정권을 수신자에게 준다.

- 사용자 화면에는 `정상`과 `주의` 두 상태만 사용한다.
- 정상 메시지는 별도 표시 없이 제공한다.
- 주의 메시지는 수신자 화면에서 본문(텍스트)을 블러 처리한다.
- 주의 메시지에는 `내용 보기`, `차단` 기능을 함께 제공한다.
- 사용자가 `내용 보기`를 실행하면 원문을 표시하고 이후 대화를 계속할 수 있다.
- 위험 판정은 수신자에게만 표시하며 발신자에게는 노출하지 않는다.

**모드별 표시 (접근성 교차 정의)**

| 모드 | 주의 메시지 처리 |
| --- | --- |
| 시각장애인 | 스크린리더는 `주의가 필요한 메시지입니다. 내용 보기를 실행하면 읽어드립니다`만 읽고, 본문은 `내용 보기` 실행 전까지 접근성 트리에서 제외한다 |
| 청각장애인 | 시각적 블러와 경고 아이콘 기본 제공 |
| 발달장애인 | 경고 문구에 쉬운 문장 원칙을 적용한다. 예: `이 메시지는 위험할 수 있어요. 볼까요?` |

### SAFE-04 내부 위험 점수

- 사용자 화면과 별도로 발신자 단위 내부 위험 점수를 누적한다.
- 정확한 점수 임계값은 모델 평가 결과에 따라 설정값으로 관리한다.
- 계정 단위 자동 제재(경고, 24시간·7일 채팅 제한)와 운영자 검토·영구 정지는 MVP 이후 계획이다.

### SAFE-05 관계 단위 자동 전송 제한 (신설)

> 같은 상대에게 위험 메시지를 반복해서 보내는 발신자는 그 상대에게 더 이상 메시지를 보낼 수 없다.

**동작 규칙**

1. 발신자 A가 수신자 B에게 보낸 텍스트 메시지가 **최근 3일 내 `주의` 판정 3회**를 누적하면, A는 B에게 추가 메시지를 전송할 수 없다.
2. 전송 제한은 A→B 관계에만 적용되며, A의 다른 대화에는 영향을 주지 않는다.
3. 수신자 B가 계속 대화를 원하면 채팅방에서 직접 제한을 해제할 수 있다.
4. 제한된 발신자 A에게는 사유를 표시하지 않고 `메시지를 보낼 수 없습니다`만 표시한다 (판정 비노출 원칙 유지).
5. 수신자 B에게는 `위험 가능성이 있는 메시지가 반복되어 전송을 제한했어요`라고 안내한다.
6. 누적 집계는 텍스트 메시지 판정만 대상으로 하며, 수신자가 `내용 보기`를 실행한 메시지도 집계에 포함한다.
7. 수신자가 제한을 해제하면 해당 관계의 누적 카운터를 0으로 리셋한다.
8. 누적 기간(3일)과 횟수(3회)는 운영 설정값으로 관리한다.

**예외 처리**

| 상황 | 처리 |
| --- | --- |
| 수신자가 제한 해제 후 다시 3회 누적 | 동일하게 재제한, 해제 가능 |
| 수신자가 발신자를 차단 | BLOCK-01이 우선 적용 (전송 제한보다 강한 조치) |

## 12. 시각장애인 모드

### VIS-01 기본 UI

- 약시와 전맹 사용자를 하나의 모드로 지원한다.
- 고대비, 큰 글자, 큰 조작 영역과 명확한 화면 구조를 제공한다.
- 글자 크기와 대비 수준을 설정할 수 있다.

### VIS-02 스크린리더

- 일반 탐색과 글 읽기는 TalkBack·VoiceOver를 기본으로 한다.
- 버튼, 입력창, 제목, 이미지 및 상태 변화에 의미 있는 접근성 이름을 제공한다.
- 자체 음성은 AI 사진 설명과 중요한 작업 상태 안내에 사용한다.
- 스크린리더와 자체 음성이 중복 재생되지 않게 한다.

### VIS-03 음성 입력

> 사용자는 말로 게시물과 메시지를 작성하고, 변환된 텍스트를 확인한 뒤 직접 게시한다.

**사용자 시나리오**

1. 녹음 버튼 실행
2. 녹음 시작 효과음 또는 안내 음성 재생
3. 사용자 발화 수집
4. `입력 완료` 실행 또는 일정 시간 무음 감지
5. 녹음 종료 안내
6. STT 결과를 텍스트 입력란에 삽입
7. 스크린리더로 변환 결과 확인
8. 필요하면 다시 말하거나 텍스트 수정
9. 사용자가 게시 버튼을 실행

**정책**

- STT는 백엔드 서버를 경유하는 OpenAI Whisper API를 사용한다 (영상 자막과 동일한 STT 모듈 재사용). 브라우저 내장 STT는 브라우저별 지원 편차로 사용하지 않는다.
- 무음 감지는 녹음만 종료한다. 음성 입력 결과를 자동 게시하지 않는다.
- `다시 말하기`를 실행하면 기존 STT 결과를 교체한다.
- 음성 데이터의 외부 전송은 17장의 고지 원칙을 따른다.

### VIS-04 상태 안내

- 녹음 시작·종료 / 변환 완료·실패 / 게시 성공·실패
- 상태는 스크린리더 알림과 효과음으로 제공한다. 진동은 지원되는 브라우저에서 보조 수단으로만 사용한다.

## 14. 발달장애인 모드 및 AI 소통 코치

### DEV-01 기본 UI

- 한 화면의 정보와 선택지를 줄인다.
- 큰 버튼, 쉬운 문장과 일관된 아이콘을 사용한다.
- 한 화면에서 하나의 주요 행동을 강조한다.
- 게시, 친구 요청 및 차단 전 명확한 확인을 제공한다.
- 그림카드 자동 생성은 제공하지 않는다.

> AI 소통 코치(COMM) 기능은 모든 UI 모드에서 사용할 수 있으며, 발달장애인 모드에서 기본 노출한다. 다른 모드에서는 해당 화면의 보조 기능으로 접근한다.

### COMM-01 쉬운 문장 변환

> 긴 게시물과 복잡한 문장을 쉬운 문장으로 바꿔 보여준다.

- 변환 결과와 원문을 전환해 볼 수 있다.
- AI 변환 결과임을 표시한다.

### COMM-02 문장 완성

> 사용자가 쓰다 만 문장을 완성한 형태로 제안한다.

- 제안 결과는 사용자가 확인하고 적용해야 한다.
- AI가 문장을 자동 입력하거나 게시하지 않는다.

### COMM-03 댓글·답장 추천

> 게시물과 대화 상황에 맞는 답변 후보를 제안한다.

- 게시물 댓글과 1:1 채팅 답장을 상황에 맞게 제안한다.
- 여러 후보 중 하나를 선택하거나 직접 작성할 수 있다.
- 선택한 문장은 전송 전에 입력창에서 확인·수정할 수 있다.

### COMM-04 대화 힌트

> 현재 대화 맥락을 바탕으로 인사, 질문, 거절 등 소통 힌트를 제공한다.

- 상대방의 의도나 감정을 사실로 단정하지 않는다.
- 코칭 레벨과 사용자 등급 기능은 제공하지 않는다.

### COMM-05 데이터 처리 원칙

- 채팅 답장 추천(COMM-03)과 대화 힌트(COMM-04)는 **사용자가 해당 기능 버튼을 직접 실행한 경우에만** 동작한다. 자동으로 대화를 분석하지 않는다.
- 기능 실행 시 해당 채팅방의 최근 N개 메시지를 외부 LLM API로 전송한다 (N은 구현 설계에서 확정, 초기값 10).
- 외부 전송의 고지·처리 원칙은 17장을 따른다.

## 17. 외부 AI 사용 및 데이터 처리 원칙

> 사용자 데이터가 어떤 목적으로 어디까지 전송되는지를 기능별로 구분해 정의하고, 가입 시 고지한다.

### 17.1 자체 처리 (외부 미전송)

| 기능 | 처리 방식 |
| --- | --- |
| 채팅 텍스트 위험 판정 (SAFE) | 자체 환경의 파인튜닝 한국어 안전 모델. 채팅 텍스트를 위험 판정 목적으로 외부에 전송하지 않는다 |
| 사용자 매칭 (MATCH) | 자체 환경의 Sentence-BERT + LightGBM |

- 채팅 사진·동영상에 대해서는 위험 판정 목적의 분석과 외부 전송을 수행하지 않는다. 접근성 목적(사진 설명·자막)의 외부 전송은 17.2를 따른다.

### 17.2 외부 AI API 사용 (가입 시 필수 고지)

| 기능 | 전송 데이터 | 전송 시점 |
| --- | --- | --- |
| AI 소통 코치 (COMM) | 사용자가 기능을 실행한 채팅방의 최근 대화 일부 | 사용자가 추천·힌트 버튼을 직접 실행한 때 |
| 사진 설명 (VISION) | 게시물·채팅 첨부 사진 | 게시 실행 시 / 채팅 사진 전송 시 |
| 자동 자막·음성 입력 (CAPTION, VIS-03) | 게시물·채팅 영상 음성 트랙, 음성 입력 녹음 | 업로드·녹음 완료 시 |

### 17.3 공통 처리 원칙

- 위 외부 전송 사실과 목적은 가입 시 필수 동의 항목 `외부 AI 데이터 처리 안내`로 고지한다.
- 외부로 전송된 데이터는 처리(분석·생성) 즉시 폐기되며, 외부 모델의 학습에 사용되지 않도록 API 제공자의 데이터 정책 옵션을 설정한다.
- 전송 데이터는 기능 수행에 필요한 최소 범위로 제한한다.
- 외부 API 장애 시의 동작은 18.3의 장애 대응 원칙을 따른다.

### 17.4 관리자 기능

- 관리자 전용 화면, 신고 접수·검토, 수동 채팅 제한 및 계정 정지는 MVP에서 제외한다.
- 신고 기능은 접수만 받고 처리하지 못하는 상태를 만들지 않기 위해, 관리자 검토 체계와 함께 후속 도입한다 (21장).
- MVP의 사용자 보호 수단은 `주의 메시지 블러(SAFE-03)`, `관계 단위 자동 전송 제한(SAFE-05)`, `사용자 차단(BLOCK-01)`, `미성년자 보호 기본값(4.5)`으로 구성한다.
- 일반 채팅 전체를 운영자에게 공개하지 않는다.

## 18. 기술 구성

### 18.1 웹 서비스

- 프론트엔드: React 기반 PWA
- 백엔드: FastAPI
- 데이터베이스: PostgreSQL
- 실시간 채팅 및 캐시: Redis
- 인증: 소셜 로그인 OAuth 2.0 (카카오 우선, 구글 지원)
- 배포: 모바일 우선 반응형 웹앱

### 18.2 AI 모듈

| 모듈 | 구성 | 운영 위치 |
| --- | --- | --- |
| AI 안심 채팅 (텍스트) | 한국어 문장 분류 모델 파인튜닝 | 자체 환경 |
| AI 사용자 매칭 | Sentence-BERT + LightGBM | 자체 환경 |
| AI 소통 코치 | LLM 기반 생성·변환 | 외부 API |
| 자동 자막·음성 입력 | Whisper STT | 외부 API (서버 경유) |
| 사진 설명 | 이미지 이해 모델 + TTS 또는 스크린리더 출력 | 외부 API |

### 18.3 설계 원칙

- AI 모듈은 기능별로 분리한다.
- AI 장애가 게시물·채팅의 기본 사용을 중단시키지 않아야 한다.
  - 자체 안전 모델 장애·지연: 친구 간 텍스트는 성능 저하 모드로 전환하여 `미분석` 표시 후 전달한다 (정상 운영의 동기 전달에 대한 예외)
  - 비친구 메시지: `안전 확인 중`으로 보류 (동기 원칙 유지)
  - 사진 설명·자막(외부 API) 장애: 채팅 사진·동영상 전송과 게시물 기본 흐름은 유지하고 `설명 없음`·`자막 없음`을 표시한다. 게시물 영상 업로드는 CAPTION-01의 예외 처리를 따른다
  - 채팅방 상단에 AI 기능 중단 안내
  - 복구 후 미분석 메시지 재분석
  - 재분석에서 주의 판정 시 소급 블러 처리와 추가 알림
  - 미분석 메시지를 `정상`으로 표시하는 행위 금지
- 장애 유형 원본 정보를 저장하지 않는다. 데이터베이스에는 사용자가 선택한 UI 모드(`ui_mode`)만 저장한다.
- AI 사용자 매칭에서 UI 모드가 외부에 노출되지 않게 한다.

## 19. 비기능 요구사항

### 19.1 접근성

- 의미 있는 HTML 구조를 사용한다.
- 키보드와 스크린리더만으로 핵심 기능을 완료할 수 있어야 한다.
- 색상만으로 상태를 구분하지 않는다.
- 터치 영역을 충분히 크게 제공한다.
- 동적 상태는 접근성 알림으로 전달한다.

### 19.2 개인정보·보안

- 계정 정보, UI 모드, 채팅 데이터와 AI 판정 데이터를 목적별로 분리한다.
- 채팅 원문 접근 권한을 최소화한다.
- AI 위험 판정 기록의 보존 기간을 운영 정책으로 관리한다.
- 외부 AI 전송 데이터는 17장의 처리 원칙을 따른다.
- 추천 결과에서 장애 유형을 노출하거나 추정 표현을 제공하지 않는다.
- 탈퇴 시 개인정보 처리는 15장의 회원 탈퇴 정책을 따른다.

### 19.3 AI 품질

- AI 안심 채팅은 정확도, 정밀도, 재현율, 오탐률과 미탐률을 측정한다. 목표값은 모델 1차 평가 후 확정하되, 위험 메시지 재현율을 오탐률보다 우선하는 방향으로 설정한다.
- 정상 문장을 과도하게 가리는지 실제 사용자 시나리오로 검증한다.
- 미성년 수신자 대상 판정은 성인 대비 낮은 임계값으로 별도 평가한다 (4.5).
- AI 소통 코치는 유해하거나 상대를 공격하는 문장을 제안하지 않아야 한다.
- 자동 자막과 사진 설명은 생성 실패를 감추지 않는다.
- AI 결과를 사실 또는 범죄 판정으로 단정하지 않는다.

## 20. 핵심 인수 조건

1. 만 14세 이상 사용자가 소셜 로그인으로 자격 인증 없이 가입할 수 있다.
2. 사용자가 세 가지 모드 중 하나를 선택하고 마이페이지에서 변경할 수 있다.
3. 시각장애인 사용자가 스크린리더만으로 피드를 탐색하고 게시물을 작성·게시할 수 있다.
4. 음성 입력과 AI 처리 결과는 자동 게시되지 않으며, 공개는 항상 사용자의 게시 실행으로만 이루어진다.
5. 게시물·채팅 사진에 AI 설명이 자동 생성되고 시각장애인에게 AI 설명임이 안내된다.
6. 영상 게시물은 업로드 즉시 자막이 생성되고, 자막 준비 완료 후 사용자가 게시를 실행해야 공개된다.
7. 발달장애인 사용자가 쉬운 문장, 문장 완성, 댓글·답장 추천과 대화 힌트를 사용할 수 있다.
8. AI 제안 문장은 사용자 확인 없이 입력되거나 전송되지 않는다.
9. 소통 코치의 대화 분석은 사용자가 기능을 직접 실행한 경우에만 동작한다.
10. 맞춤 친구 추천이 Sentence-BERT·LightGBM 결과를 사용하고 UI 모드를 노출하지 않는다.
11. 미성년 사용자에게 성인 발신 비친구 메시지 요청이 기본 차단되고, 미성년-성인 상호 추천이 제외된다.
12. 비친구 메시지는 분석 완료 후 요청함에 분리 표시되며, 수락 전 전송 제한이 적용된다.
13. 텍스트 메시지는 자체 안전 모델의 분석 완료 후 전달되며, 주의 판정 메시지는 블러 처리되고 수신자가 직접 열람 여부를 결정할 수 있다.
14. 동일 수신자 대상 최근 3일 내 주의 판정 3회 누적 시 해당 관계의 메시지 전송이 자동 제한되고, 수신자가 해제할 수 있다 (해제 시 카운터 리셋).
15. AI 탐지만으로 채팅 원문이 운영자에게 공개되지 않는다.
16. 사용자를 차단하면 게시물, 프로필, 친구 요청, 채팅과 추천에서 상호 제외된다.
17. 관리자 화면 없이 사용자 기능 전체를 시연할 수 있다.
18. 채팅 사진·동영상은 안전 분석 없이 전송됨이 표시되고 사진 설명·자동 자막이 제공되며, 미성년-성인 채팅에서는 전송이 제한된다.
