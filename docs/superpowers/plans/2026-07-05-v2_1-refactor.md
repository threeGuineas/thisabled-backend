**Goal:** `docs/ThisAbled_기능명세서_v2_1.md`(SSOT) 정합으로 백엔드 스키마·API·문서를 전면 갱신한다.

**Architecture:** v3 baseline 스키마를 한 번에 깔고(빅뱅), 문서 4종을 갱신한 뒤, 도메인별로 API를 구현한다. 인증은 소셜 OAuth 전용(mock 제공자), SAFE·MATCH는 별도 모델 서버 HTTP 경계(mock 컨테이너), 채팅은 REST + WebSocket(Redis pub/sub). 설계 근거: `docs/superpowers/specs/2026-07-05-v2_1-refactor-design.md`.

**Tech Stack:** Python 3.11 · FastAPI 0.111 · SQLAlchemy 2 async · PostgreSQL 15 · Redis 7 · Alembic · python-jose · APScheduler(신규) · httpx

## Global Constraints

- 모든 작업은 `feature/v2_1-refactor` 브랜치에서. 태스크 완료 = pytest 그린 + 1커밋(Conventional Commits, `[S1]` 태그).
- 테스트 실행: `docker compose exec -T app pytest -q` (파일 단위: `docker compose exec -T app pytest tests/test_auth.py -q`)
- PK·FK 전부 UUID (`gen_random_uuid()` 서버 default). 예외: `forbidden_nicknames.id`(기존 serial 유지), 복합 PK 테이블.
- `ui_mode`: `visual|hearing|developmental` 3종만. 'default' 없음. 만 나이·연령대·미성년 여부는 저장하지 않고 `birth_date`로 계산.
- 명세 고정값: 만 14세 이상 가입 / 미성년=만14~18 / 닉네임 2~12자 / bio ≤300자 / 태그 ≤10개 / 사진 ≤3장·영상 1개 / 영상 ≤3분·≤200MB / vision 20회·일+5회·분(게시물·채팅 합산) / caption 5회·일 / 재시도 ≤2회 / SAFE-05 3일·3회 / 재가입 제한 30일 / 드래프트 TTL 24h / COMM 최근 메시지 N=10 / SAFE 타임아웃 2초. 전부 Settings 환경변수로.
- 에러 응답 `{"detail": ...}`. 비노출 원칙: 차단·연령 정책 사유는 `요청을 보낼 수 없는 상대입니다`, SAFE-05 제한 발신자에겐 `메시지를 보낼 수 없습니다`만.
- 외부 API(OpenAI)·모델 서버는 테스트에서 항상 DI fake로 대체. 실호출 금지.
- 코드 주석·에러 메시지는 한국어, 기존 파일 스타일 유지.
